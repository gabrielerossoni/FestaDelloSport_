## Troubleshooting ğŸ”§

### Il sito non si connette al backend

1. Verifica che `CONFIG.API_BASE_URL` in `js/config.js` sia corretto
2. Controlla che il backend sia in esecuzione
3. Verifica CORS nel backend (dovrebbe essere giÃ  configurato)
4. Controlla la console del browser per errori

### Le immagini non si caricano

1. Verifica i percorsi relativi delle immagini
2. Assicurati che la cartella `assets/` sia presente
3. Controlla che i file esistano nel repository

### Service Worker non funziona

1. GitHub Pages serve i Service Worker solo su HTTPS
2. Verifica che il file `sw.js` sia nella root del progetto
3. Controlla la console del browser per errori
4. Pulisci la cache del browser se necessario

### Tailwind CSS non funziona

1. Assicurati di aver eseguito `npm run build:css`
2. Verifica che `css/style.css` sia stato generato
3. Controlla che `tailwind.config.js` sia configurato correttamente
4. Verifica che `css/input.css` contenga le direttive Tailwind

### Errori di rate limiting

Se ricevi errori "429 Too Many Requests":
- Il backend limita le richieste per proteggere da spam
- Limiti: 5 prenotazioni/minuto, 10 feedback/minuto
- Attendi qualche minuto prima di riprovare

### Database non funziona

1. Verifica che il file `backend/festa_sport.db` esista
2. Controlla i permessi di scrittura sulla cartella `backend/`
3. Esegui backup regolari con `python backend/backup_db.py`
