# Configurazione - Festa dello Sport

## Setup Iniziale

1. **Configurazione Backend URL**
   - Apri `js/config.js`
   - Modifica `API_BASE_URL` per l'ambiente di produzione:ascript
     API_BASE_URL: 'https://tuo-backend-url.com'
     ## Variabili d'Ambiente

### Frontend
- **CONFIG.API_BASE_URL**: URL del backend API
  - Sviluppo: `http://localhost:3001`
  - Produzione: da configurare in `js/config.js`

### Backend
- **PORT**: Porta del server Flask (default: 3001)
- **DB_PATH**: Percorso del database SQLite (default: `backend/festa_sport.db`)

## Deploy

### Frontend (GitHub Pages)
1. Push del codice su GitHub
2. Abilita GitHub Pages nelle impostazioni del repository
3. Il sito sarà disponibile su `https://tuonome.github.io/FestaDelloSport_/`

### Backend

1. Assicurati che Python 3.x sia installato
2. Installa le dipendenze:
   pip install flask flask-cors
   3. Avvia il server:
   cd backend
   python backend2.py
   4. Per produzione, usa un servizio come:
   - Heroku
   - Railway
   - Render
   - DigitalOcean App Platform

## File di Configurazione

- `js/config.js` - Configurazione frontend (URL API)
- `backend/backend2.py` - Configurazione backend (PORT, DB_PATH)
- `index.html` - Meta tag, titoli, descrizioni

## Troubleshooting

### Il sito non si connette al backend

- Verifica che `CONFIG.API_BASE_URL` in `js/config.js` sia corretto
- Controlla che il backend sia in esecuzione
- Verifica CORS nel backend (dovrebbe essere già configurato)

### Le immagini non si caricano

- Verifica i percorsi relativi delle immagini
- Assicurati che la cartella `assets/` sia presente

### Service Worker non funziona

- GitHub Pages serve i Service Worker solo su HTTPS
- Verifica che il file `sw.js` sia nella root del progetto
- Controlla la console del browser per errori
